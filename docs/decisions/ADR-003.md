# ADR-003: Why OrbStack over Docker Desktop

**Status:** Accepted
**Date:** 2026-04-04

## Context

The hub runs 10+ containerized services (media stack, document management, automation, chat UI). On macOS, the two main options for running Docker containers are Docker Desktop and OrbStack. Both run a Linux VM under the hood since macOS doesn't support containers natively.

## Decision

**Use OrbStack as the container runtime.**

### Why

1. **Resource efficiency.** OrbStack uses significantly less CPU and memory at idle compared to Docker Desktop. On a machine that also runs Ollama with 70GB+ models loaded, every gigabyte of overhead matters. OrbStack's lightweight VM design leaves more unified memory available for inference.

2. **Startup speed.** OrbStack starts in seconds and containers launch near-instantly. Docker Desktop's VM initialization is noticeably slower, especially after a reboot — relevant for a machine that serves as a 24/7 home server.

3. **Native macOS integration.** OrbStack integrates cleanly with macOS — containers appear in Finder, volumes are accessible, and the menu bar app is minimal. Docker Desktop's UI is heavier and more intrusive.

4. **Docker Compose compatibility.** OrbStack is fully compatible with Docker Compose files — no changes needed. The `docker` and `docker compose` CLI commands work identically. Switching back to Docker Desktop would require zero config changes.

5. **Network handling.** OrbStack handles port forwarding and container networking cleanly on macOS, including the container-to-host communication that services like Radarr and Prowlarr need to reach Ollama on the host.

### Alternatives considered

- **Docker Desktop:** The incumbent. Functional but heavier — uses more RAM at idle, slower cold starts, and the licensing model adds complexity (free for personal use, paid for commercial). For a home lab, the licensing is fine, but the resource overhead isn't.
- **Lima + nerdctl:** Lightweight and open source, but requires more manual setup and lacks OrbStack's polished macOS integration. Better suited for developers comfortable with CLI-only workflows.
- **Podman:** Daemonless container engine. Good in theory, but Docker Compose compatibility has rough edges, and the macOS experience is less mature than OrbStack.

## Consequences

- All container management uses standard `docker` and `docker compose` commands — no vendor lock-in
- If OrbStack introduces breaking changes or becomes unmaintained, migrating to Docker Desktop is a drop-in swap
- OrbStack is a commercial product with a free personal tier — if pricing changes, Docker Desktop or Lima are fallbacks
- The team of one (me) benefits from the lower cognitive overhead of a tool that stays out of the way
