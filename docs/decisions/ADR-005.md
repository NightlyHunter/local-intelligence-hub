# ADR-005: Why External SSD → UniFi NAS Migration Path

**Status:** Accepted
**Date:** 2026-04-04

## Context

The media stack (Phase 2) needs persistent storage for downloaded movies, TV shows, and music. The UniFi UNAS-4 NAS is planned for September 2026. The project needs a storage strategy that works now and transitions cleanly to the NAS later.

### Storage options evaluated

| Option | Cost | Capacity | Redundancy | Available now |
|---|---|---|---|---|
| Internal 2TB SSD | $0 (included) | ~1.5TB free after OS + models | None | Yes |
| 1TB Samsung T7 External SSD | $0 (already owned) | ~1TB | None | Yes |
| Buy NAS now | ~$840+ (NAS + 2 drives) | 8TB+ (RAID 1) | Yes | Budget constraint |

## Decision

**Use the 1TB external SSD as a "hot library" now. Migrate to the UniFi UNAS-4 (2×8TB IronWolf, RAID 1) when it arrives in September 2026.**

### Why this path

1. **Internal SSD is reserved for performance-critical data.** The 2TB internal SSD holds macOS, Ollama models (~70GB primary + fallbacks), container configs, and Paperless-ngx indexes. Media files would compete with model loading for I/O bandwidth and consume space needed for future phases.

2. **The external SSD is already available at zero cost.** The Samsung T7 is USB 3.2 — fast enough for media serving. Jellyfin streaming 4K HDR peaks at ~100 Mbps; the T7 delivers ~1,000 Mbps. No bottleneck.

3. **1TB is enough for a curated library, not an archive.** At 1080p, it holds ~50-80 films. At 4K, ~15-20. This is a "hot library" — keep favorites, rotate the rest. The NAS removes this constraint with 8TB+ of redundant storage.

4. **The UNAS-4 is the right NAS for this architecture.** At $379, it's the cheapest option that fits the UniFi ecosystem. It doesn't run Docker or apps — but it doesn't need to. The Mac Studio is the compute engine; the NAS is pure storage over SMB/NFS. More capable NAS units (Synology DS925+ at ~$640, UGREEN DXP4800 at ~$600) would be paying for compute capacity that duplicates the Mac Studio.

5. **RAID 1 → RAID 5 expansion path.** Starting with 2×8TB in RAID 1 (8TB usable) allows adding 2 more drives later for RAID 5 (24TB usable). The UNAS-4 handles this transition automatically via its "Basic Protection" tier. All drives must be the same size.

### Migration plan

1. **Pre-NAS:** All media on `/Volumes/T7/media/`. Hardlinks keep downloads and library on same volume (no double storage).
2. **NAS arrives:** Set up RAID 1, create SMB/NFS shares, copy media from T7 to NAS.
3. **Post-migration:** Update Jellyfin, Radarr, Sonarr, Lidarr mount paths from T7 to NAS share. T7 becomes overflow/travel drive.
4. **Backup targets updated:** rsync jobs pointed at NAS instead of external SSD.

## Consequences

- Media has no redundancy until September 2026 — acceptable since media is re-downloadable
- 1TB capacity requires active curation (delete what you've watched, keep favorites)
- The external SSD must stay connected and mounted at `/Volumes/T7` — if it disconnects, all Arr services lose their library paths
- Migration to NAS will require a maintenance window to copy data and update all service configs
- Critical data (Paperless docs, Actual Budget DB) is backed up to the external SSD via rsync — this data is NOT re-downloadable and gets NAS backup priority
