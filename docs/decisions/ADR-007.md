# ADR-007: Why Jellyseerr for Non-English Title Discovery

**Status:** Accepted
**Date:** 2026-04-04

## Context

The media stack (Radarr, Sonarr) searches for content by title. When searching for a movie or TV show by its Russian or Ukrainian title, Radarr and Sonarr often fail to find it — torrent indexers catalog content under English titles, while the user is searching in their native language. This creates a gap: you know the movie as "Брат" but the indexer lists it as "Brother."

Radarr and Sonarr search TMDb (The Movie Database) for metadata, but their search is primarily English-oriented. The user would need to manually look up the English title, then search again in Radarr — a friction point that defeats the purpose of automation.

## Decision

**Add Jellyseerr as the request management frontend for Radarr and Sonarr.**

### Why

1. **TMDb localization solves the language gap.** Jellyseerr searches TMDb with full localization support — users can search in Russian, Ukrainian, or any language TMDb supports. It resolves the localized title to a TMDb ID, then passes that ID (not the title string) to Radarr or Sonarr. The Arr service uses the TMDb ID to find the correct English-titled release on indexers. The language barrier disappears.

2. **Request workflow adds structure.** Jellyseerr provides a request/approval workflow. When sharing the media stack with others (Phase 5), users can request content through a clean UI rather than accessing Radarr/Sonarr directly. Requests can require approval before downloading — useful for managing disk space on the 1TB external SSD where every gigabyte counts.

3. **Better discovery UX.** Jellyseerr's interface is designed for browsing and discovering content — trending movies, popular shows, recommendations. Radarr and Sonarr are designed for library management, not discovery. Different tools for different jobs.

4. **Jellyfin integration.** Jellyseerr integrates with Jellyfin to show what's already in the library, preventing duplicate requests. Users see "Available" or "Requested" status directly in the discovery UI.

### Alternatives considered

- **Overseerr:** The original project that Jellyseerr forked from. Overseerr is Plex-focused; Jellyseerr adds Jellyfin support. Since this stack uses Jellyfin, Jellyseerr is the correct fork.
- **Manual TMDb lookup:** Search TMDb manually, find the English title, paste it into Radarr. Works but adds friction to every non-English search — exactly the kind of manual step automation should eliminate.

## Consequences

- One more container to run and maintain (port 5055)
- Jellyseerr becomes the primary interface for requesting content — Radarr/Sonarr are managed "behind the scenes"
- TMDb API dependency — if TMDb is down, Jellyseerr can't search (but Radarr/Sonarr can still be used directly as a fallback)
- Per-user accounts enable controlled access for Phase 5 friends without exposing the Arr stack directly
